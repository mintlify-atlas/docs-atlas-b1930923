---
title: "Quickstart"
description: "Get EdgeCaster running and serving RTSP streams in under 10 minutes."
---

This guide walks you through installing EdgeCaster on a Raspberry Pi 5, connecting it to your Rhombus organization, and streaming a camera to a VMS or NVR.

<Info>
Before starting, make sure you have a Raspberry Pi 5 running Ubuntu Server 24.04 with internet access. See [hardware requirements](/hardware-requirements) for the full list.
</Info>

<Steps>
  <Step title="Clone and install">
    SSH into your Raspberry Pi and run the installer:

    ```bash
    git clone https://github.com/RhombusSystems/edgecaster-stream-converter.git
    cd edgecaster-stream-converter
    sudo bash scripts/install.sh
    ```

    The installer handles everything: FFmpeg, MediaMTX, Python, Node.js, Nginx, the edgecaster system user, all directories, frontend build, and systemd services. It is safe to re-run.

    When complete, EdgeCaster will be running and accessible on port 80.
  </Step>

  <Step title="Open the web UI">
    Open a browser on your local network and navigate to:

    ```
    http://edgecaster.local
    ```

    or use the device's IP address directly:

    ```
    http://<device-ip>
    ```

    You'll see the first-run setup screen asking for your Rhombus Org API Key.
  </Step>

  <Step title="Enter your Rhombus Org API Key">
    Your Org API Key can be found in the [Rhombus Console](https://console.rhombussystems.com) under **Settings → API Keys**.

    Enter the key and click **Save**. EdgeCaster will immediately contact the Rhombus API to discover all cameras in your organization.

    <Warning>
      Keep your API key secure. It is stored in `/etc/edgecaster/config.yaml` (readable only by the `edgecaster` system user) and is never logged or exposed in API responses.
    </Warning>
  </Step>

  <Step title="Enable cameras">
    The camera list shows all discovered cameras with their name, location, and online status.

    Toggle the switch next to any camera to start its RTSP stream. EdgeCaster will:
    1. Request a Secure Raw Stream URL from the Rhombus API
    2. Start an FFmpeg process to pull the HTTPS H264 stream
    3. Publish it to MediaMTX as a named RTSP path

    You can enable up to **10 cameras** concurrently.

    <Note>
      The RTSP path slug is derived from the camera name. "Front Door" becomes `front_door`, "Warehouse Bay 2" becomes `warehouse_bay_2`.
    </Note>
  </Step>

  <Step title="Connect your VMS or NVR">
    Once a camera shows **running**, point your VMS, NVR, or video application to:

    ```
    rtsp://<device-ip>:8554/<camera-slug>
    ```

    Real examples based on camera names:

    ```
    rtsp://192.168.1.100:8554/front_door
    rtsp://192.168.1.100:8554/warehouse
    rtsp://192.168.1.100:8554/parking_lot
    ```

    Test playback with VLC:

    ```bash
    vlc rtsp://192.168.1.100:8554/front_door
    ```
  </Step>
</Steps>
